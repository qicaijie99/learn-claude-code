# 1. 先说结论（最重要）

你想要的其实不是：

> ❌ 直接用 Anthropic 或 DeepSeek SDK

而是：

> ✔ 自己做一层 “中间 runtime（统一 block 层）”
> ✔ 或直接用 LiteLLM / LangGraph 这种封装

---

# 2. 核心矛盾（你现在困惑的点）

| 系统        | 输出结构                             |
| --------- | -------------------------------- |
| Anthropic | blocks（TextBlock / ToolUseBlock） |
| OpenAI    | JSON（message + tool_calls）       |
| DeepSeek  | OpenAI兼容 JSON                    |
| Gemini    | function calls + text            |

👉 它们**根本不是同一种 runtime**

---

# 3. 你想要的目标其实是这个：

```text
统一结构：
[
  {"type": "text"},
  {"type": "tool_use"},
  {"type": "tool_result"}
]
```

👉 也就是你自己定义一个：

> 🧠 “Block Runtime Layer”

# 4. 标准工程方案（推荐）

## ✔ 方案 A：LiteLLM（最推荐）

直接解决你问题：

```python
from litellm import completion

response = completion(
    model="anthropic/claude-3-5-sonnet",
    messages=messages
)
```

换模型：

```python
model="openai/gpt-4o"
model="deepseek/deepseek-chat"
model="gemini/gemini-pro"
```

👉 同一套接口

---

### 但问题：

❌ LiteLLM 统一的是 “messages”
❌ 不是 block

# 5. 真正你想要的方案（关键）

你要的是：

> ✔ Anthropic block runtime + 多模型兼容

👉 正确做法是：

# 🧱 自己加一层 “Block Adapter Layer”

---

## ✔ 架构长这样：

```text
            ┌──────────────┐
            │   LLM API    │
            │ Claude/OpenAI│
            └──────┬───────┘
                   ↓
        ┌────────────────────┐
        │  Response Adapter   │
        │ (convert to blocks) │
        └────────────────────┘
                   ↓
        ┌────────────────────┐
        │  Unified Blocks     │
        │ Text / Tool / etc   │
        └────────────────────┘
                   ↓
        ┌────────────────────┐
        │   Agent Runtime     │
        │ (你的 loop)        │
        └────────────────────┘
```

---

# 6. 关键设计：Adapter（核心）

你要做的是：

## ✔ OpenAI → Block

```python
def openai_to_blocks(resp):
    return [
        TextBlock(text=resp["choices"][0]["message"]["content"])
    ]
```

---

## ✔ DeepSeek → Block（因为它是 OpenAI 兼容）

```python
def deepseek_to_blocks(resp):
    return openai_to_blocks(resp)
```

---

## ✔ Anthropic → Block（原生）

```python
def anthropic_to_blocks(resp):
    return resp.content
```

---

# 7. 统一接口（你最终会得到）

你 agent 只认：

```python
response.content  # 永远是 blocks
```

不管底层是谁。
